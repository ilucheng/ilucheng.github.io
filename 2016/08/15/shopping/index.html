在商店用手机支付时，您的钱是这样带着忧伤的走向老板的钱包的

Aug 15, 2016 | 金融支付 | Hits
文章目录

感谢大家对《发红包时，您的钱是这样带着爱心走向小伙伴的钱包的》的支持，文章发表后，收到小伙伴给削土豆的私信，问到当我们用手机支付宝去商店扫码付款的时候，我们的钱是怎么带着忧伤走向老板的钱包的。削土豆仔细考虑，这其中，要有Pos机运营商，商户，支付条码参与，所以流程与手机发红包相似而又不同，下面由削土豆用葛二蛋去花店买花的例子来为您解释，为什么二蛋要买花，请看下图^V^。


花店想用Pos机进行收款，需要到Pos机运营商申请Pos，Pos 机运营商在部署Pos 到商店的时候，会做如下重要的事情，简单列举

  1. 给商户分配一个商户ID来确定商户的唯一性。
  2. 商户 ID 绑定要部署的 Pos 机的 SN（序列号）。这样通过这台 Pos 机就可以知道是哪个商户在操作了。
  3. 绑定支付通道，如支付宝，微信等。
  4. 商户 ID 绑定商家支付帐号。
  5. 选择连接方式（直连or间连）

这里，我们给花店部署了一台 Pos 机 （间连）SN：98765，分配的商户 ID 为 A0001，绑定支付通道：支付宝，支付宝帐号：xiaotudou

葛二蛋去花店，千挑万选最终以 999 元买了束“蓝色妖姬”，拿出手机准备付钱，打开支付宝，选择支付方式“余额宝”，展示条码“123456”（条码在支付宝离线的时候，也是可以生成并进行付款的），店员在互联网 Pos 机上输入金额999元，扫手机条码，这时，Pos 机会发送以下重要数据到 Pos 机运营商

  1. 扫到的条码：12345
  2. 输入的金额：999
  3. Pos 机的 SN：98765
  4. 商户 ID：A0001
  5. 支付通道：支付宝
    Pos 机运营接收上述数据之后，进行数据分析
  6. 得到支付通道：支付宝
  7. 通过商户 ID 得到商户的支付宝 ID：xiaotudou

分析结束，汇总一句话，生成 12345 条码的用户通过支付宝转账999元到支付宝账号xiaotudou。

然后Pos运营商将数据发送到支付宝那里，支付宝收到数据之后，进行数据分析

  1. 通过条码 12345 可以得到是葛二蛋的支付宝生成的。

分析结束，汇总一句话，葛二蛋的支付宝转账999元到支付宝账号xiaotudou，执行操作，葛二蛋的 999 元就转到 xiaotudou 的帐号里了。

转账成功后，支付宝通道葛二蛋转账成功，通道Pos机运营商转账成功，Pos 机运营商通知 Pos 机转账成功，Pos 机显示“付款成功“，打印小票，交易成功。
